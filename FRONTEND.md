# LabSync AI Frontend

A minimal Next.js 14 (App Router) UI that surfaces structured events generated by the backend.

## Stack
- Next.js 14 (App Router, server components)
- TypeScript
- Tailwind CSS via `@tailwindcss/postcss`

## Structure
```
frontend/
├─ app/
│  ├─ layout.tsx     # Global shell + nav
│  ├─ page.tsx       # Landing hero
│  ├─ events/page.tsx
│  └─ components/
│     ├─ EventCard.tsx
│     └─ EventList.tsx
├─ types/event.ts    # Shared typings
├─ tailwind.config.ts
└─ app/globals.css   # Tailwind + design tokens
```

## Environment
Create `frontend/.env.local` (or reuse root `.env.example` values):
```
NEXT_PUBLIC_API_BASE_URL=http://localhost:4000
```

## Running
```bash
cd frontend
npm install
npm run dev
# open http://localhost:3000
```

`/events` uses a server-side fetch to `NEXT_PUBLIC_API_BASE_URL + /api/events`, so ensure the backend is running first.

## Components
- **EventCard** — renders a single event with title, tag, schedule, and confidence label.
- **EventList** — grid wrapper that handles empty state messaging.

## Styling
Tailwind powers the layout. Base colors/fonts live in `app/globals.css` so the UI matches the product brief (clean, minimal, blue accent).

## Extending
- For pagination/filters, extend `app/events/page.tsx` to pass query params to the backend
- Introduce client components or SWR if you need live updates
- Add mutation forms that call the Event Management API for edits/deletes
